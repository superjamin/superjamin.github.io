java基础-List

发表于 2020-04-08 | 分类于 java基础

list主要实现有ArrayList,LinkedList,Vectory和Stack

ArrayList

构造函数

默认容量10，在没有指定容量的情况下，第一次放进元素时设置

DEFAULTCAPACITY_EMPTY_ELEMENTDATA 用以区分是否从无参构造进来，并在第一次放元素时设置容量10

当数组满了，就扩容1.5倍

Java中set，list和queue都实现了iterable接口，代表这些集合对象都可以用迭代器来对集合中的数据进行迭代。在list中，他有一个自己独有的迭代器，listiterator，和iterator相比，它还支持往前迭代。

arrayList判断数据是否相同是equals（），是返回true，比如remove（）：

remove操作最后调用System.arraycopy方法，将被删除元素之后的元素复制到原数组，和前面的拼接，将最后位置置null，等待gc处理

至于get(int)、set(int,E)，直接通过数组下标即可

1.内部用数组实现，默认容量是10，每次扩容1.5倍，调用的是Arrays.copyOf(旧数组，新容量)

2.是有序的，通过索引直接操作数据

3.是线程不安全的

LinkedList

LinkedList同时实现list和Deque接口，所以它也同时具有了list和queue的特点，它既有索引，是一个list，同时也是一个双向队列，所有它的操作方式也比较多样，list和queue的都可以用

内部类Node对象最为重要，它组成了LinkedList集合的整个链表，分别指向上一个点、下一个结点，存储着集合中的元素；

get

先判断索引有没有越界，然后再将索引和集合大小的1/2比较，若小于，则从头开始循环遍历到index处，返回该位置的node，反之，则从后往前遍历

remove

根据下标和对象删除元素的remove方法，最后调用的都是unlink（）方法

add(e)方法调用的linkLast(e),其实就是加入一个新node，对前节点（若不为null）的后指针改为加入node，后节点的前指针（若不为null）改为加入node

clear()方法就是遍历集合，把每个node的都置null，确保gc回收成功

Vectory

Vector的用法和arraylist基本一致，从jdk1.0开始就有的一个集合。

*Vector是线程安全的，因此性能上没有arraylist好。

Stack

Stack是Vector的一个子类，模拟了栈的数据结构

总结：

1.如果需要遍历集合，ArrayList和Vector的性能通过随机访问的方式比较好（用数组存储数据），

LinkedList用迭代器比较好（链表存储）

2.LinkedList执行插入删除操作性能比较好（链表），ArrayList和Vector用数组实现，要动态扩容，耗费时间

3.Vector保证线程安全

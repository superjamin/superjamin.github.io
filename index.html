作为java开发，相信NPE是大家经常碰到的异常，它总是不经意的出现，着实让人头疼。java8引入了一个新类——java.util.Optional，使用它可以帮助我们解决各种null引发的问题。让我们一起来看看吧。

创建Optional对象

Optional利用泛型，对类做了一层封装。创建Optional对象不是通过new，因为该类的两个构造函数是私有的

创建Optional对象有三种方式：

1.声明一个空的Optional

​ Optional optCar = Optional.empty();

2.依据一个非空值创建Optional

​ Optional optCar = Optional.of(car);

​ 但是当传入的car为null时，会立刻抛出NullPointerException。

3.可接受null的Optional

​ Optional optCar = Optional.ofNullable(car);

Ｍａｐ是以键值对（ｋｅｙ／ｖａｌｕｅ）的形式来保存数据，ｋｅｙ和ｖａｌｕｅ具有映射关系。主要的ｍａｐ实现有HashMap,LinkedHashMap,Hashtable,WeakedHashMap和TreeMap

HashMap

默认容量16，默认加载因子0.75f，hashmap中支持插入null作为key，key是唯一的，插入重复的key时，后插入的value会代替之前的value。我们还注意到插入的顺序和我们输出的顺序不一致，这是因为HashMap在插入key时，会通过计算hashcode来自己确定存储位置，所以是无序的。HashMap还可以以集合的形式返回所有的key，通过HashMap的keySet()可以对map的对象进行迭代。通过get（Object key）方法，可以获取到对应的value。

LinkedHashMap

LinkedHashMap在HashMap的基础上，通过链表来维护插入的key的顺序，保证放入和取出的顺序是一致的。
因为需要维护插入的顺序，所以性能上没有HashMap好。

Hashtable

Hashtable在实例化的过程中就对table对象完成了初始化过程，默认容量为11，加载因子也是0.75f。Hashtable和HashMap的用法基本相同，区别在于Hashtable是线程安全的，不支持null作为key。

WeakedHashMap

WeakedHashMap和HashMap的用法相同，区别的是WeakedHashMap中放的是弱引用

TreeMap

   对照TreeSet, TreeSet内部通过红黑树来存储数据，要求插入的对象都实现compator接口，依次来保证内部元素处于有序状态。TreeMap也是通过红黑树实现了内部对象的排序。
在实例化时，通过内部类直接重写compare（）方法，可以实现定制排序的效果。
*除非需要map中的对象总是处于有序状态，此时选择TreeMap，其他时候就应该尽量使用其他集合，因为排序会额外消耗时间。

当多个Key对象，通过计算得到了同一个index的情况，就产生hash冲突

equals方法

重写equals方法

​ 为什么要重写equals方法？我们知道Object类中的equals方法是用==比较两个对象内存地址是否相同，以此来判断两个对象是否相等。但是真实情况是，我们需要从逻辑上去判断对象是否相等。

hashCode()方法

hashcode（）方法获取hash码，我们把它看成一个int整数，它的作用是确定对象在散列结构中的位置。

为什么重写equals（）方法要重写hashcode（）方法

如果对象被放入散列结构中，那么hashcode（）就会起作用。比如HashMap，当我们放入对象时，根据key的hashcode来确定存入的位置，如果存入的位置已经有了元素，就通过equals（）方法去比较key，若返回true则覆盖原来的key-value。

由此我们可以看出equals（）和hashcode（）的关系：

  • 两个对象equals（）返回true时，它们的hashcode值需要相等

  • 两个对象的hashcode值相等，它们通过equals比较不一定相等（hash冲突）

当重写equals方法时，必须遵守它的通用约定。

1.自反性：对于任何非空引用x，x.equals(x)必须返回true

2.对称性：对于任何非空引用x、y，如果且仅当x.equals(y)返回true时，y.equals(x)必须返回true

3.传递性：对于任何非空引用 x、y、z，如果 x.equals(y) 返回 true，y.equals(z) 返回 true，则 x.equals(z) 必须返回 true。

4.一致性： 对于任何非空引用 x 和 y，如果在 equals 比较中使用的信息没有修改，则 x.equals(y) 的多次调用必须始终返回 true 或始终返回 false。

5.对于任何非空引用 x，x.equals(null) 必须返回 false

针对以上特性，编写equals方法的步骤如下：

1.用==运算符检查是否为该对象的引用，若是则返回true；

2.使用 instanceof 运算符来检查参数是否具有正确的类型。若不是则返回false；

3.经过instance of的筛选，那么能直接将参数转换为正确的类型。

4.对于类中的每个「重要」的属性，请检查该参数属性是否与该对象对应的属性相匹配。

数组定义

数组是相同类型数据的有序集合。数组中的每一个元素都可以通过下标来访问。数组是引用类型的数据。

具有几个基本特点：

  1. 数组的长度是确定的，一旦被创建，不可被改变
  2. 数组元素必须是相同类型的
  3. 数组类型可以是任何类型（基础数据类型和引用类型）

数组的初始化

数组使用分为两部分，先是声明一个数组变量，这时在栈中创建一个引用，初始化后在堆中分配空间。只读成员length是数组对象的一部分（事实上这是唯一一个可以访问的字段或方法），表示数组可以存储多少个元素。

1.静态初始化

int [] arr = new int [] {1,2,3};

或

int [] arr = {1,2,3};

2.动态初始化

int [] arr = new int [3];

当数组初始化只声明了数组的大小，系统会给数组中的元素分配默认值：

整型数据（byte、short、int、long）默认值是0；

浮点型数据（float，double）默认值是0.0；

字符型数据（char）默认值是0对应的字符编码；

布尔类型（boolean）默认值是false；

引用类型数据（数组、类和接口）默认值是null；

数组的内存管理

我们知道java的内存分两种，栈内存用来存基本数据类型和引用数据类型的引用变量（创建和销毁的成本低），堆内存用来存实例对象（创建和销毁的成本高），栈内的引用指向堆内的具体对象。

​ 举个例子，在数组内存放一个book对象，实际存放的是对book对象的引用，操作修改数组内book对象的值，其实都是修改堆中的book对象值。
